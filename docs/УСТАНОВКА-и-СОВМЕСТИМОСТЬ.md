# Установка на существующий сайт, совместимость и захардкоженные данные

Документ для программиста-партнёра. Описывает: что содержит наш набор изменений, как накатить его на уже развёрнутый сайт без поломок, что меняется в поведении всего сайта, и какие данные захардкожены и требуют замены.

Все наши файлы помечены комментарием `STAGING:` вверху — по нему легко находить наши правки.

---

## 1. Что это за изменения

Один набор правок поверх текущего кода (Laravel 12 / PHP 8.4, Blade, Tailwind, mcamara/laravel-localization). Он делает три вещи:

1. **Новые SEO-страницы** (аддитивно): 10 бесплатных инструментов (`/tools`), 7 VIN-проверок (`/vin-check`), 9 сравнений (`/compare`), страница отзывов (`/reviews`), Data Sources (`/data-sources`) и статические/юридические страницы.
2. **Редизайн** (визуальный, влияет на весь сайт): новый первый экран, шрифт **Inter**, брендовая палитра переведена с teal на графит + оранжевый акцент.
3. **Мультиязычность нового контента**: en/ru/uk/pl/kk/es через языковые файлы.

## 2. Совместимость с развёрнутым сайтом

### 2.1. Аддитивно и безопасно (ничего существующего не ломает)

- Новые контроллеры: `ToolController`, `VinCheckController`, `ComparisonController`, `ReviewController`, `StaticPageController`.
- Новый хелпер `app/Support/HomepageStats.php`.
- Новый конфиг `config/reviews.php`.
- Новые вьюхи в `resources/views/tools/`, `vin-checks/`, `comparisons/`, `static/`, `reviews.blade.php`.
- Новые языковые файлы: `lang/{en,ru,uk,pl,kk,es}/{home,compare,vincheck,tools,nav}.php`.
- Новые маршруты в `routes/web.php` (см. п. 3.3).
- **Новых миграций БД нет.** `HomepageStats` читает существующую таблицу `orders` (колонки `vin`, `report_type`, `status`, `created_at`) — проверьте, что они есть в проде.
- **Новых зависимостей composer/npm нет.**

### 2.2. Меняет поведение ВСЕГО сайта — согласовать обязательно

1. **`tailwind.config.js` — перекрашена палитра `primary`.** Была teal (`#073b4c`…), стала графит/почти-чёрный (`#141518`…) + добавлен `acc` (оранжевый). Это влияет на **каждую** страницу, где используется `primary-*` — включая блог, панель, дашборд, страницы авторизации. Либо принимаете новый бренд-цвет глобально, либо переопределяете `primary` обратно и используете `acc`/`gray` только на новых страницах.

2. **Шрифт → Inter** (был Figtree). Меняется в `tailwind.config.js` (`fontFamily.sans`) и подключается в `resources/views/header.blade.php` и `resources/views/layouts/app.blade.php` (bunny.net). Тоже глобально.

3. **`resources/views/header.blade.php`** — добавлены ссылки на Inter и оранжевая полоса сверху (`<div class="h-1 bg-acc-500">`). Появляется на всех страницах, использующих этот header.

4. **`resources/views/footer.blade.php`** — перестроен в 4 колонки (Info + инструменты/проверки/сравнения из реестров контроллеров). **Важно:** ссылки на юридические страницы переведены с БД-маршрута `/page/{slug}` на новые статические маршруты (`terms`, `privacy`, `refund`) + добавлены `cookie` и `data-sources`. Если в проде реальные юр-тексты лежат в БД (`Page` через `/page/{slug}`), решите: (а) оставить наши статические страницы (тогда контент из БД больше не показывается в футере), либо (б) вернуть в футере ссылки на `/page/{slug}` и удалить наши статические страницы/маршруты.

5. **`resources/views/index.blade.php`** — переверстан первый экран + подключены новые партиалы. Логика VIN-формы, checkout и bundles сохранена (id/имена/JS не тронуты).

### 2.3. Гварды окружения (проверить перед go-live)

- **noindex**: `app/Http/Middleware/ForceNoIndex.php` + мета в header ставят `noindex` только на env `local/development/staging`. На проде страницы индексируются — убедитесь, что `APP_ENV=production`.
- **Платежи**: в TEST-режиме через env — на проде должны быть боевые ключи.
- **hCaptcha**: тестовый режим только на `local`.

## 3. Как накатывать (пошагово)

### 3.1. Скопировать/смёржить файлы

Наши файлы — это: новые файлы из списка «untracked» + изменённые вьюхи/контроллеры/конфиги/lang + `tailwind.config.js` + `routes/web.php` + `header/footer/index`. Все помечены `STAGING:`.

> ⚠️ **Не делайте `git add -A`.** В рабочем дереве, кроме наших правок, есть «фантомные» изменения из-за окончаний строк (CRLF/LF) на файлах, которые мы **не** трогали: `docker/entrypoint.sh`, `lang/*/auth.php`, `mail.php`, `pagination.php`, `passwords.php`, `validation.php`, `railway.json`, `.dockerignore` и др. У них `git diff --stat` показывает **0 изменений строк** — это чистый шум от переносов строк. Коммит `docker/entrypoint.sh` с CRLF **сломает запуск контейнера**. Стейджите только осознанно нужные файлы (`git add <путь>` поимённо или папками `resources/views/tools` и т.п.), проверяя `git diff --cached`.

### 3.2. Команды деплоя

```bash
composer install --no-dev -o          # зависимости не менялись, но на всякий случай
npm ci && npm run build               # ОБЯЗАТЕЛЬНО: новый CSS (Inter, палитра, новые классы)
php artisan config:clear
php artisan route:clear
php artisan view:clear
php artisan cache:clear
```

`npm run build` — критично: без пересборки `public/build` стили поедут (старый CSS не содержит `acc-*`, `gray-900/10`, Inter и т.д.).

### 3.3. Новые маршруты (routes/web.php)

Дублируются на верхнем уровне и в группе `{locale}` (`.locale`-имена). Проверьте, что не конфликтуют с существующим catch-all `/{slug}` или `/page/{slug}`:

```
/tools, /tools/{tool}, /tools/api/decode-vin, /tools/api/recalls
/vin-check, /vin-check/{check}
/compare, /compare/{competitor}
/reviews
/money-back-guarantee
/data-sources
/terms-and-conditions, /privacy-policy, /refund-policy, /cookie-policy
```

### 3.4. Проверка после установки

Прогнать (в т.ч. переключая язык): главная, `/tools`, `/tools/vin-decoder`, `/compare`, `/compare/carfax`, `/vin-check`, `/vin-check/accident-history`, `/data-sources`, `/terms-and-conditions`, футер на любой странице. Если языковой файл сломан — страница отдаст 500 (в этой среде нет PHP для линта, поэтому смоук-тест обязателен).

### 3.5. Откат

Оригиналы всех изменённых при редизайне файлов лежат в `_demo1_bak/` (в корне проекта). `git checkout -- <файл>` тоже вернёт исходник.

## 4. Захардкоженные данные — что заменить

Отсортировано по важности. Всё это сейчас «заглушки/статизм», которые нужно связать с реальными данными.

### 4.1. Последние 10 VIN / «недавно проверенные» (соц-пруф)

`app/Support/HomepageStats.php`:
- `recentReports()` читает реальные `orders` в проде. **Но** `sampleReports()` — это захардкоженный список из 10 выдуманных авто, который показывается, когда заказов нет (на staging). В проде с заказами он не используется — но проверьте.
- Даже для реальных заказов `'vehicle' => ''` (марка/модель не хранится в заказе) и `records` — **псевдо-число** (`pseudoRecords()` = `7 + crc32(vin) % 34`, детерминированное, не настоящее). **Надо:** сохранять декодированную марку/модель и реальное число записей в момент заказа, иначе колонки «Vehicle»/«Records» неинформативны.
- `purchases24h()`: при 0 реальных заказов возвращает **выдуманное** `31 + (dayOfYear % 23)`. В проде с заказами — реальное. Уберите фолбэк, когда пойдут заказы.

### 4.2. Отзывы

`config/reviews.php` — **захардкожены**: рейтинг `5.0`, `count => 32`, `sales => 226`, и ~22 текста отзывов, скопированных с вашего Etsy. Показываются как testimonials + ссылка на Etsy (намеренно **без** self-serving Review/AggregateRating schema — Google его не засчитывает). **Надо:** обновлять вручную при изменениях на Etsy, либо подключить лицензированный виджет (Trustpilot / Google Business Profile) для «звёзд» в выдаче. Значение `5.0 · 32 reviews` также продублировано в eyebrow первого экрана.

### 4.3. Цены

- **Наша цена `$3.00`** захардкожена в разметке и текстах: первый экран (рельса + подзаголовок), `home-price-compare`, страницы сравнений/`compare.php`, vin-check/`vincheck.php`, тексты инструментов. Реальная продажная цена — **динамическая** (`$carfaxPrice`). Если поменяете цену — `$3.00` в маркетинге надо править вручную (или завести единую конфиг-константу и подставлять её).
- **Цены конкурентов** ($44.99 Carfax, $24.99 AutoCheck, $19.99 EpicVIN, $9.99 и т.д.) захардкожены в `ComparisonController::ITEMS`, в `home-price-compare.blade.php` и в текстах `lang/*/compare.php` / `vincheck.php`. Приблизительные, помечены «approximate» — держите в актуальном состоянии (и в контроллере, и в языковых текстах, где число зашито в предложение).

### 4.4. Schema.org на главной (⚠️ нестыковка)

`resources/views/index.blade.php`, `$productSchema`:
- `aggregateRating` = **4.8**, `reviewCount` = **66** — **противоречит** заявленным на странице `5.0 / 32`. Свести к одному значению.
- `offers`: `lowPrice => 1.5`, `highPrice => 3` — сверить с реальными ценами.

### 4.5. Организация / реквизиты

`resources/views/header.blade.php`, `$organizationSchema`: `legalName => 'FOP Yakovenko Igor Eduardovich'`, `taxID => '2167301354'`, адрес в Харькове, `sameAs` (Etsy/YouControl/Trustpilot). Проверить актуальность для прода.

### 4.6. Прочее

- E-mail поддержки `support@cheapvinreport.email` — в футере, money-back и всех статических страницах.
- Маркетинговые заявления: «256-bit SSL», «~2 min delivery», «100% money-back», «24 hours», «226+ sales» — убедиться, что соответствуют действительности.
- Пример отчёта: `public/images/Example.pdf` (2018 Lexus ES 350) — используется в `home-sample-report` и в шапке-навигации.
- Модели оплаты конкурентов и их фичи в `ComparisonController::ITEMS` (`model`, `fair`) — приблизительные.

## 5. Статус мультиязычности

Переведено на все 6 языков (через языковые файлы, контроллеры/вьюхи отрефакторены под `__()`):
- Новые блоки **главной** — `lang/*/home.php`.
- **Сравнения** (9) полностью — `lang/*/compare.php`.
- **VIN-проверки** (7) полностью, включая тексты JS-воронки — `lang/*/vincheck.php`.
- **Инструменты**: реестр (названия, описания, `<title>`/meta, хаб) + тизер на главной — `lang/*/tools.php`.
- **Футер**: заголовки колонок, «Все…», ссылки — `lang/*/nav.php`.

**Ещё на английском** (осознанно, самый объёмный/низкоприоритетный кусок): внутренние тела 10 инструментов (подписи полей калькуляторов + методология + строки их JS) и тексты контент-страниц (Data Sources, terms/privacy/refund/cookies/money-back). Их названия и meta уже переведены. Отзывы намеренно оставлены на английском.

## 6. Резюме рисков в одну строку

Главный риск совместимости — **глобальная смена палитры `primary` и шрифта**: они меняют вид всего сайта, а не только новых страниц. Второй — **не забыть `npm run build`** и **не коммитить CRLF-фантомы** (особенно `docker/entrypoint.sh`).
